## 3. Bug đã fix

### P1-2 · `findById` admin loại nhầm soft-deleted

**Trước**: Admin gọi `GET /api/products/:id` hoặc `GET /api/posts/:id` cho item đã soft-delete → trả **404** → không xem được thùng rác, không pre-validate trước khi restore.

**Fix**:

- `be/src/models/product/product.service.ts` · `findById(id, { includeDeleted? })`
- `be/src/models/post/post.service.ts` · `findById(id, { includeDeleted? })`
- Admin controller endpoint `@Get(':id')` (cả product & post) gọi với `{ includeDeleted: true }`. Public endpoint vẫn ẩn (mặc định `includeDeleted=false`).

**Verify**: tạo → soft-delete → admin `GET /:id` → 200 với `deletedAt` ISO ✅

### P1-3 · Post `publishedAt` không tự đóng dấu

**Trước**: PATCH `status: PUBLISHED` mà DTO không kèm `publishedAt` → field giữ `null` → bài viết "đã publish" nhưng không sort theo thời gian được, public feed lọc `publishedAt: { lte: now }` cũng loại.

**Fix** (`post.service.ts` `update()`):

```ts
const current = await this.findById(id);
let publishedAt: Date | undefined;
if (dto.publishedAt) publishedAt = new Date(dto.publishedAt);
else if (dto.status === PostStatus.PUBLISHED && !current.publishedAt) publishedAt = new Date();
```

Chỉ stamp khi DRAFT → PUBLISHED lần đầu; lần publish sau giữ giá trị cũ; admin vẫn override được qua DTO.

**Verify**: tạo DRAFT → PATCH PUBLISHED (không truyền publishedAt) → GET → `publishedAt` là ISO mới ✅

## 4. P1-1 CSRF — quyết định defer (có rationale)

**Hiện trạng**: `CsrfGuard` + `CsrfService` đã có (`be/src/common/guards/csrf.guard.ts`, `be/src/services/csrf/`), nhưng **không controller nào apply** và **FE không gửi `X-CSRF-TOKEN`**.

**Đánh giá rủi ro**:

- Cookie auth dùng `sameSite: "strict"` (`be/src/common/utils/cookies.ts`) → browser **không gửi cookie auth** với cross-site request → CSRF vector đã bị chặn ở tầng browser.
- `secure: true` ở production, `httpOnly: true`, `signed: true`.

**Tại sao chưa apply guard**:

1. Bật `CsrfGuard` cần đồng thời:
   - `GET /api/csrf/token` endpoint (chưa có)
   - FE axios interceptor fetch/cache token + attach header (chưa có)
   - Apply guard vào tất cả mutation controller (site-config, products, posts, categories, users, banners, media, reviews)
2. Triển khai vội = break toàn bộ admin mutation. SameSite=strict đã là phương tiện chính, CSRF token là defense-in-depth.

**Khuyến nghị (next iteration)**:

- Tạo `csrf.controller.ts` với `GET /csrf/token` (set signed cookie `csrf_secret` + trả `csrfToken`)
- FE `lib/axios.ts`: request interceptor — với method ∈ {POST, PUT, PATCH, DELETE} và chưa cache token → call `/csrf/token` → cache module-level → attach `X-CSRF-TOKEN`
- Apply `@UseGuards(JwtAuthGuard, RolesGuard, CsrfGuard)` cho admin mutation controller
- Production: thu hẹp CORS allowlist (hiện reflective allow-all để dev) — `origin: [process.env.FRONTEND_URL]`
